<!-- Plugin changelog -->
## 3.0.5
**Capabilities -- 核心能力增强**

* **Slash Command 支持 Skill 扩展**：增强 Slash Command 机制，支持直接调用与编排 Skill 能力，提升指令式交互效率与复杂任务的执行灵活性。
* **优化 `@` 引用文件 Prompt 生成规则**：重构引用文件时的 Prompt 拼接策略，提升上下文相关性与结构清晰度，减少冗余信息干扰，增强模型理解准确性。


**Stability & Improvements -- 稳定性与优化**

* **`@` 搜索文件与文件夹升级为基于 Ripgrep 实现**：底层搜索能力升级为基于 Ripgrep 的高性能检索引擎，显著提升大规模代码仓库中的搜索速度与结果完整性，解决此前搜索缓慢与结果缺失问题。
* **搜索与引用链路稳定性优化**：优化文件检索与引用流程中的异常处理与边界判断，提升在大仓库与复杂目录结构下的整体稳定性与一致性。


**Bug Fixes -- 缺陷修复**

* **修复输入框选择文件时误选问题**：解决在文件选择输入框中输入 `.` 或空格时自动选中第一条记录的问题，提升文件引用操作的准确性与交互体验。
* **增强第三方模型 API 兼容性**：完善对 Kimi、GLM 等模型官方 API 的适配能力，提升与 LiteLLM 的兼容性与稳定性，增强多模型接入场景下的可靠性。


## 3.0.4

**Capabilities -- 核心能力增强**

* **新增 `ResolveClassName` 工具（Java 项目）**：支持根据上下文智能解析与补全类名（含全限定名推断），提升跨文件分析与代码理解的准确性与效率。
* **新增 `ListImplementations` 工具（Java 项目）**：支持查找接口或抽象类的所有实现类，提升面向接口编程场景下的结构分析与代码导航效率。
* **Agent Checkpoint 能力实现**：引入 Agent 执行过程中的 Checkpoint 机制，支持阶段性状态保存与恢复，增强复杂任务链路的可追踪性与可控性。

* **Git Commit Message AI 自动生成**：支持基于变更内容自动生成规范化 Commit Message，减少手动编写成本，提升提交效率与一致性。

## 3.0.3

**Capabilities -- 核心能力增强**

* **新增 `/slash` 命令支持**：支持通过 Slash Command 快速触发操作与上下文能力，提升交互效率。
* **Java 项目能力增强**：新增 `ReadClass` 工具，支持直接读取与解析 Java Class 结构，优化代码理解与分析体验。
* **Markdown 表格渲染优化**：提升表格解析与展示稳定性，改善复杂表格场景下的可读性。

**UX -- 交互与体验优化**

* **EditorSelectionTag 机制优化**：

    * `AddToJarvisChat` 触发后自动移除 EditorSelectionTag，同时**保留编辑器选中状态**。
    * 新建 Chat 时，不再自动注入当前编辑器选中内容，减少误上下文。
    * 打开文件时不恢复历史选中内容，避免上下文污染。
* **工具调用流程优化**：调整校验顺序，**优先检查工具是否存在，再校验参数合法性**，减少模型重复调用无效工具的情况。

**Stability & Bug Fixes -- 稳定性与修复**

* **Windows Bash 进程清理修复**：修复 Bash 工具停止后 Node 子进程仍持续运行的问题，优化 Windows 下派生进程终止策略。
* **连续错误计数修复**：修复 AI Message 为空时未计入连续错误的问题，避免长时间等待后才触发 API 次数异常报错。
* **任务状态流转修复**：修复提交任务时无法从「未开始」进入「开发中」状态的问题，确保流程正确推进。
